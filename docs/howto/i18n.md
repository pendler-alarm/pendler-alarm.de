# i18n

## Overview

The project uses `vue-i18n` with German (`de`) and English (`en`) message files.

Relevant files:

- `src/i18n/index.ts`
- `src/i18n/messages/de.ts`
- `src/i18n/messages/en.ts`
- `scripts/check-i18n.ts`

## Add or change translations

1. Add the new key in `src/i18n/messages/de.ts`.
2. Add the same key in `src/i18n/messages/en.ts`.
3. Keep the object structure identical in both files.
4. Use descriptive nested keys such as `auth.google.message.scope`.

## Use translations in Vue components

Use `useI18n()` inside `<script setup>` and call `t()` in script or template.

```vue
<script setup lang="ts">
import { useI18n } from 'vue-i18n';

const { t } = useI18n();
</script>

<template>
  <h1>{{ t('about.title') }}</h1>
</template>
```

## Use translations in TypeScript modules

For plain TypeScript files outside Vue components, use `translate()` from `src/i18n/index.ts`.

```ts
import { translate } from '@/i18n';

const title = translate('about.title');
```

## Locale handling

- The initial locale is resolved in `src/i18n/index.ts`.
- Supported locales are derived from the `messages` object.
- The selected locale is stored in `localStorage` using the key `app-locale`.
- German is the fallback locale.

## Check i18n consistency

Run the i18n check before merging translation changes:

```bash
npm run check:i18n
```

The script validates:

- missing keys in `de` or `en`
- unknown keys used in source files
- unused translation keys
- possible hardcoded UI text outside the i18n files
